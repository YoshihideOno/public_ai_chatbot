# E2E認証フローテスト設計書

## 1. 概要
本ドキュメントは、AI Chatbot Platformの認証関連E2Eテストシナリオを定義する。
Playwrightを用いたエンドツーエンドテストを想定している。

## 2. テストシナリオ

### 2.1. ユーザー登録・ログインフロー

#### シナリオ1: 新規ユーザー登録からログインまで
1. ログインページにアクセス
2. 「アカウント登録」タブをクリック
3. 有効なテナント情報と管理者情報を入力
4. 登録ボタンをクリック
5. 登録成功メッセージを確認
6. ログインページに遷移することを確認
7. 登録した認証情報でログイン
8. ダッシュボードに遷移することを確認

#### シナリオ2: 既存ユーザーのログイン
1. ログインページにアクセス
2. 有効な認証情報を入力
3. ログインボタンをクリック
4. ダッシュボードに遷移することを確認

#### シナリオ3: パスワードリセットフロー
1. ログインページにアクセス
2. 「パスワードをお忘れですか？」リンクをクリック
3. 登録済みのメールアドレスを入力
4. 送信ボタンをクリック
5. 成功メッセージを確認
6. メール内のリンクをクリック（モック）
7. 新しいパスワードを入力
8. パスワードリセットを確認
9. 新しいパスワードでログイン

### 2.2. エラーハンドリング

#### シナリオ4: 無効な認証情報でのログイン試行
1. ログインページにアクセス
2. 無効な認証情報を入力
3. ログインボタンをクリック
4. エラーメッセージが表示されることを確認
5. ログインページに留まることを確認

#### シナリオ5: バリデーションエラーの表示
1. ログインページにアクセス
2. 無効なメールアドレス形式を入力
3. ログインボタンをクリック
4. バリデーションエラーメッセージが表示されることを確認

## 3. テスト実行手順

```bash
cd frontend
npm run test:e2e
```
